# Statistical Edge Engine

## Overview

The **Statistical Edge Engine** is the analytical core of QuantStudio. It quantifies trading edges, detects market regimes, and ranks assets based on statistically sound metrics, allowing you to focus on high-probability setups aligned with current market conditions.

## Key Concepts

### Regime Detection

Every analysis cycle starts by classifying the current market environment:

- **Trending Markets**: Direction is sustained; +ADX > 25; price channels are sloping.
  - **Momentum strategies** thrive (e.g., breakout pullbacks).
  - **Mean-reversion** fails because price rarely retraces to moving averages.

- **Sideways (Ranging) Markets**: ADX ≤ 25; Bollinger Bands width < 4% of average true range.
  - **Mean-reversion setups** work best (RSI extremes, fades from upper/lower bands).
  - **Trend-following attempts** result in whipsaws and drawdowns.

All regime detection uses rolling window statistics without look-ahead bias; each ticker stands alone to prevent contagion.

## Technical Formulas

### Directional Movement (ADX)

```
+DI = (High<sub>t</sub> - High<sub>t-1</sub>) / ATR<sub>14</sub>
-DI = (Low<sub>t-1</sub> - Low<sub>t</sub>) / ATR<sub>14</sub>
DX = abs((+DI - -DI) / (+DI + -DI))
ADX = moving_avg(DX, 14 days)
```

### Regime Thresholds

| Regime       | ADX        | 20-day Slope | Notes           |
|-------------|------------|--------------|-----------------|
| Trending    | > 25      | > 0.001      | Strong slope    |
| Sideways   | 15–25     | ~0           | Flat bands      |

## Edge Score Weights

edge_score = 0.40 momentum + 0.30 (1 - drawdown) + 0.30 regime_alignment

### Normalized Momentum

ROC_14 = (Close<sub>today</sub> - Close<sub>14</sub>) / Close<sub>14</sub>
normalized = percentile(ROC_14, all_window_ativos)

### Drawdown Protection

max_drawdown_win = minimum(rolling(close, 30 days) - rolling(max, 30 days)) / rolling(max, 30 days)
normalized = 1 - max_drawdown_win

### Regime Alignment

fit = 1 if strategy matches regime; else 0.1 (weak or abstain).

## Decision Trees

### Signal Branch

If edge > 80 → **Strong Buy** (high momentum, low drawdown, aligned regime).
If 70 ≤ edge ≤ 80 → **Buy** (neutral).
If 50 ≤ edge < 70 → **Wait** (moderate).
If edge < 50 → **Sell / Avoid** (low momentum, high DD, mismatched regime).

---

## Why This Matters

### Precision

Limit decisions to only assets with quantified edges, avoiding whipsaws and revenge trading.

### Consistency

Same formula every run → no recency bias or emotional drift. Results are reproducible JSON for audit or API use.

### Scalability

Process hundreds of tickers per second in batch mode, surfacing only the top decile for human review.

### Documentation

All criteria are plain text in `config.json` and TUI menus; no hidden parameters.

## Integration

Edge Engine underpins three central TUI commands:

- **Run on Single Ticker** → `edge_engine.py capp.py of single asset
- **Run on Universe** → `batch rank of list
- **Edge Discoverer** → `micro-setup flag fits regime

Dependencies are isolated; you can override any Ford engine subclass in Python without breaking the CLI.
