# 8. NEWS ANALYSIS

## Overview
Il modulo **News Analysis** sfrutta LLM AI per analizzare notizie di mercato in tempo reale, identificare eventi ad alto impatto e generare report strutturati. Utilizza feed RSS configurabili per aggregare notizie da fonti globali (es. Google News) e valuta l'impatto su settori/ticker specifici.

> **⚠️ Nota**: Per funzionare, il modulo richiede un endpoint LLM attivo (configurato in `config/config.json`).

---

## 8.1 Generate Today's High-Impact Analysis (≥6)

### Funzionalità
- **Filtro ad Alto Impatto**: Mostra solo notizie con `impact_score ≥ 6`, ordinate per rilevanza decrescente.
- **Visualizzazione Interattiva**:
  - Colori dinamici basati su `impact_score` (rosso per ≥8, giallo per ≥6).
  - Dettagli per ogni notizia: `title`, `published_at`, `impact_reason`, `tickers` associati.
- **LLM-Driven**:
  - Genera un riassunto di mercato (`overview`) con settori impattati (`key_themes`).
  - Spiega le ragioni dietro ciascuno score.

### Esempio Output
```plaintext
NEWS ANALYSIS - Europe
──────────────────────
[Market Summary - 2026-05-14]
Sentiment: Neutral con rischio ribassista sui titoli tech per dati economici USA.

[Settori Impattati]: technology, energy

╔════════════════════════╗
║ Impact Score: 8      ║  ← Rosso (impatto critico)
╠════════════════════════╣
║ Titoli coinvolt...   ║
╚════════════════════════╝
```

## Configurazione

### Feed RSS
I feed di notizie sono configurati in `config/config.json`, nella sezione `news`:
```json
"news": {
  "Europe": ["https://news.google.com/rss?hl=it&gl=IT&ceid=IT:it"],
  "America": ["https://news.google.com/rss?hl=en-US&gl=US&ceid=US:en"]
}
```
> **🔧 Aggiunta**: Puoi estendere i feed con qualsiasi fonte RSS (es. Reuters, Bloomberg). Esempio:
> ```json
> "Asia": ["https://example.com/rss/feed"]
> ```

### Parametri LLM
Il modulo invia prompt strutturati al LLM utilizzando l'API definita in:
```json
"llm": {
  "api_url": "https://test-cx-api.giorgioarmani.tech/_v1/llm/proxy/v1/chat/completions",
  "api_key": "sfl-token-llm-very-secret"  ← Configura nel file `.env` (non nel repo!)
}
```

## Workflow Tecnico
1. **Fetch**:
   Il modulo `news/fetcher.py` scarica le notizie dal RSS configurato per la regione (`news_region`).
2. **Analisi LLM**:
   `news/analyzer.py` invia le notizie all'endpoint LLM con un prompt strutturato (vedi [Prompt Template](#prompt-template)).
3. **Output**:
   - Salva un report JSON in `/news_analysis_{region}_{date}.json`.
   - Mostra i risultati in TUI (opzione 8.1 del menu principale).

### Prompt Template (LLM)
```text
Analizza la seguente notizia per impatto di mercato, fornendo:
1. impact_score (da 1 a 10).
2. impact_reason (breve spiegazione).
3. tickers (lista di simboli azionari coinvolti).
Notizia: "{title}" - {content_snippet}
```
