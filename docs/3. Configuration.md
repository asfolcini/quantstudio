# Configuration

## Purpose

The Configuration menu allows users to customize QuantStudio behavior and integrations without editing code. All settings are persisted in `config.json` and immediately reflected in UI, analytics, and external feed behavior.

## Menu Location

From the main TUI, choose option **9 – Configuration** to open the secondary menu:

```
Configuration Options
----------------------
1 – View current settings
2 – Change data provider
3 – Change report language
4 – Change news region
5 – Change LLM API
6 – Test LLM
0 – Back
```

---

## 1. View Current Settings

### What It Does

Renders a read-only summary table of active settings:

| Key                | Value                     |
|--------------------|---------------------------|
| LLM API URL        | https://api.openai.com/v1 |
| LLM API Key        | sk-proj-*** (masked)      |
| Data Provider      | yfinance                  |
| Report Language    | english                   |
| News Region        | Europe                    |

### Why It Matters

- Immediate audit trail for team collaboration or troubleshooting calls. No need to dig into JSON files.
- Verify values after creation or updates, especially sensitive fields like API keys.

---

## 2. Change Data Provider

### What It Does

Displays a yellow message: “Only YAHOO provider is supported at this time.”

After acknowledging, returns to Configuration menu.

### Why It Matters

- Future-proofing placeholder. When Binance, Interactive Brokers, or Polygon integrations land, the same UI slot will offer a multi-provider picker.

---

## 3. Change Report Language

### What It Does

1. Lists available languages from `core/ui_mappings.py`:

```
[1] English
[2] Italiano
[0] Back
```

2. Updates `config.json` field `report_language`.

3. Immediate feedback: “Report language updated to Italiano.”

### Why It Matters

- Removes language barrier for non-native English traders.
- Localized metric abbreviations or date formats appear in TUI tables.

---

## 4. Change News Region

### What It Does

1. Lists regions defined in `REGION_MAP`:`
```
[1] United States
[2] Europe
[3] Italy
…
[0] Back
```

2. Updates `config.json` field `news_region`.

3. Immediate feedback: “News region updated to Italy.”

### Why It Matters

- Filters or formats any market-specific news or events displayed in future TUI panels.
- Prevents region-locked content errors.

---

## 5. Change LLM API

### What It Does

Prompts for two strings:

```
Enter new LLM API URL [https://api.openai.com/v1/chat/completions] >
Enter new LLM API Key [] > ************
```

- Validates key non-empty.
- Saves both to `config.json` under `llm.api_url` and `llm.api_key`.
- Success message: “LLM API settings updated successfully.”

### Why It Matters

- Let users swap fog for openLLM, Mistral, or custom deployments.
- No code edits required; pure configuration lever.

---

## 6. Test LLM

### What It Does

1. Sends a lightweight POST request carrying:
```json
{
  "messages": [{"role": "user", "content": "hi there, are you there ?"}]
}
```

2. Displays a multi-panel result:

**Success Panel**
```
✓ Success
Response: "Hello! How can I assist today?"
Latency: 0.45 s
```

**Error Panel**
```
✗ Failed: Request timed out (10s)
```

3. Errors handled separately:
- Timeout → explicit timeout message.
- Connection refused → "Could not connect" message.
- Unexpected JSON structure → "Unexpected format" message.

### Why It Matters

- Pre-flight check before running Hedge Discoverer with AI insights.
- Immediate feedback eliminates silent failures in setups.

---

## Under the Hood

- **Storage**: All values persisted in human-readable `config.json`.
- **Security**: Keys are masked at display; never echoed to command line or stored in repo.
- **Extensibility**: Each menu option mapped to `update_config(key, value)` from `core/config_loader.py`.

## Real-World Usage Example

1. Move from `INT` to `FTSE_MIB` universe.
2. Change report language to `en`.
3. Point LLM endpoint to local Ollama instance -> verify with `Test LLM`.
4. Start `Hedge Discoverer` → inputs now use user’s locale and accessible AI model.

## Summary

Configuration is the throttle you adjust while the engine runs. Minor tweaks cascade through pipelines; remaining consistently smooth and reproducible.
