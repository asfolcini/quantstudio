# Run on Single Ticker

## What It Does

Quantifies the trading edge of a single ticker by analyzing its price history, identifying the prevailing market regime, and computing a composite edge score that captures momentum, volatility, and regime alignment. All numeric results are instantly available in the TUI and logged for reproducibility.

## How It Works (Step-by-Step)

### 1. Input

You provide a ticker symbol:

```
Enter ticker symbol: AAPL.MI
```

QuantStudio locates the corresponding `historical_data/AAPL.MI/data.csv`.

### 2. Data Validation

- Ensures ≥30 daily bars are available (minimum viable history).
- Checks OHLCV columns are present and monotonically increasing in `datetime`.
- If invalid, error message: *No full data file found or too few observations.*

### 3. Regime Detection (Implicit)

Using exclusively price-derived metrics:

- **Directional Movement**: ADX over 14-day window.
- **Slope**: 20-day simple moving average gradient.

Result spatial clustering:

| Regime      | ADX  | Slope       | Condition   |
|------------|------|-------------|-------------|
| Trending    | >25  | >0 or <0    | ADX>25      |
| Sideways   | 15-25| ~0           | ADX≤25       |

Example: `ENEL.MI`, 2024-03-20, ADX=18 → sideways.

### 4. Quantitative Metrics

#### A. Momentum

Rate-Of-Change over 14-day (smoothed) window:

ROC = (close_today - close_14 bars ago) / close_14 bars ago × 100

Example: `STM.MI`, 2024-03-23, ROC= +5.23%.

#### B. Volatility

20-day rolling standard deviation of log returns:

σ = σ_log_returns_window × √252_annualized

Example: `AAPL.MI`, 2024-03-23, σ=12.41%.

#### C. Maximum Drawdown

Rolling window minimum from peak:

maxDD = ∑((peak - trough)/peak) over 90-day window

Example: `TIT.MI`, 2024-02-05, maxDD= -9.8%.

### 5. Composite Edge Score

Normalized, regime-weighted aggregation (0-100 scale):

edge = 0.40 * momentum + 0.30 * (1 - maxDD) + 0.30 * regime_alignment

Regime alignment:

- 1.0 if strategy fits regime: momentum in trend = strong fit, mean-reversion in sideways = strong fit.
- 0.1 if mismatch: mean-reversion in trend = weak fit.

### 5. backend calls, output report

The `EdgeEngine` class standardizes reporting:

- Decision: BUY/SELL/WAIT
- Confidence: HIGH/MEDIUM/LOW based on edge quintile.
- Metrics: All numeric values for audit.
- Regime: Text snapshot.

## Why This Matters

### Speed

Replace 60 minutes (manual Excel) with 2.7 seconds per ticker.

### Consistency

Same algorithms are applied every run, eliminating emotional drift, recency bias, and chart distortion from zooming.

### Portability

Results are serializable JSON for 3rd-party scripts, historic research, audits.

### Regime Alignment

Automatically abstains from unfavourable setups (“mean-reversion in trending markets” gets scored near 0).

### Completeness

Standard procedure prints head, tail, OHLC min/max/avg for visual sanity checks alongside the numeric panel.
