# Data Management

## Purpose

The **Data Management** section lets you control and maintain the data pipeline that powers all analysis. It handles everything from fetching raw market data, cleaning corporate actions, filling gaps, standardising file structures, and grouping assets into logical baskets for batch analysis.

## Why It Matters

- **Garbage In, Garbage Out**: No qualitative insights or profitable edges can emerge from broken data. Data Management ensures reliable inputs.
- **Historical Tracking**: Maintain auditable data dumps for compliance or research.
- **Efficiency**: Avoid repeated manual downloads and transformations for the same assets.

## Objectives

1. Populate the tickers you wish to track (e.g. FTSE MIB stocks, 500 US tech stocks, 10 crypto pairs).
2. Keep local data in sync with market feeds (e.g. Yahoo Finance) with minimal human intervention.
3. Clean raw data of gaps, spurious spikes, corporate actions, and exchange anomalies.
4. Group assets into logical universes (e.g. "FTSE MIB", "Energy", "Crypto Large Cap ") for batch analysis and relative strength ranking.

## 1. List All Tickers

### What It Does

Scans the `historical_data` folder and outputs a summary table listing every ticker currently controlled. Each row displays:

- **Ticker**: Canonical symbol (e.g. `AAPL.MI`, `ENEL.MI`).
- **Name**: Full company name (auto-fetched: "iPhone Inc", "Enel Spa").
- **Exchange**: Market (e.g. Milan `.MI`, London `.L`).
- **First Date**: Oldest data point.
- **Last Date**: Newest data point.

### Why It Is Useful

- **Quick audit**: Before running analysis, ensure your candidates exist and are complete.
- **Spot gaps**: Missing tickers or incomplete dates indicate failed fetches needing repair (`Update All data`).
- **Inventory management**: Track which universes you have curated.

## 2. Add New Ticker

### What It Does

- Asks for one or more comma-separated ticker symbols (upper-case, country suffix accepted: GOOG, AAPL.MI, BRKB, IBM).
- Validates the format and uniqueness; rejects duplicates.
- Triggers a fresh download from the configured provider (default: Yahoo Finance). Cleaned data is stored in `historical_data/GOOG/data.csv`.

### Syntax

```
Tickers: AAPL.MI,ENEL.MI,STM.MI
```

### Why It Is Useful

- **Authorize new candidates** for weekly / monthly analysis cycles.
- **Quick-list addition** of temporary watch-lists (e.g. sector rotations, themes).

### Real World Example

```
You: AAPL.MI,MSFT.MI
QuantStudio: SUCCESS added 2/2 tickers.
```

## 3. Remove Ticker

### What It Does

- Deletes the ticker directory under `historical_data` (including `data.csv`, metadata, sanitised files).
- Dhreads any universe that contained the ticker (keeps universe file intact but removes entry).

### Why It Is Useful

- **Delisted stocks** (e.g. BKR.MI): Clean up stale symbols.
- **Sector rotation**: Remove sectors no longer tracked.
- **Temporary focus**: Streamline universe for faster `Run on Universe` sweeps.

## 4. Update All Data (Most Used)

### What It Does

Orchestrates a bulk refresh of **every** ticker in `historical_data`. Per-ticker actions:

- **Full Sync**: Delete existing CSV, reload entire history from Yahoo Finance (gap-less corporate-action-adjusted bars).
- **Delta Sync**: Append market days missing between last local bar and yesterday (intraday bars removed).

A progress spinner and tally are displayed live during the multi-threaded fetch/cached write loop.

### Why It Is Useful

- **Fix data drift**: Silent provider changes (splits, holidays) break spreadsheet models silently; QuanteStudio re-aligns immediately.
- **New methodology testing**: Re-work entire universe with new motif without corrupting live files.
- **After clone**: Refresh daily on GitHub Actions to auto-commit a "data layer" for team shared use.

### Real World Example (FTSE MIB, April 2024)

```
[Recap]
Tickers processed: 40
Successful: 38
Failed: 2 (A2A.MI corporate action, URSS.MI delisted).
```

## 5. Inspect Data

### What It Does

Displays the last 10 OHLCV rows and full metadata for any single ticker.

- Missing values, negative volumes, extreme price jumps are instantly visible.
- Metadata includes provider hierarchy, timezone, dividend-adjustment status.

### Why It Is Useful

- **Debug gems**: Did post-market get cleaned yesterday as expected?
- **Reconciliation**: Confirm `close` matches broker statement before authorising batch order.

## 6. Manage Universes

### What It Does

Group tickers into logical baskets stored in `config/universes.json`. Each universe has a friendly name and list of symbols to habitually compare (e.g. FTSE MIB baskets, sector breaks).

### Why It Is Useful

- **Batch workflows**: `Run on Universe` uses this basket to compute relative metrics across all members.
- **Relative strength**: Rank which stocks are outperforming their own sector or index.
- **Universal programmatic access**: Any script can run `load_universes()` and iterate consistently.

### Options

- List: Show every defined universe and ticker counts.
- Create: Add new basket with empty members.
- Add ticker to universe: Append one or multiple symbols to existing basket.
- Remove ticker from universe: Prune without deleting data.
- Delete universe: Remove entire basket file.

### Real World Example

```
Universe "FTSE MIB" contains 40 tickers.
Universe "Tech MIB" contains 5 tickers: AAPL.MI, MSFT.MI, NVDA.MI, INTC.MI, AMD.MI.
```
