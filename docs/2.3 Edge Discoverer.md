# Edge Discoverer

## What It Does

Scans your universe for marketplace anomalies that align with prevailing market conditions (regime). It filters thousands of hypothetical setups using regime detection, backtested score criteria, and AI-insights to present only high-expected-value setups ranked by edge.

## Step-by-Step Process

### 1. Input

Select either:

- Single ticker (e.g., `AAPL.MI`).
- Entire universe (e.g., `FTSE_MIB`).

### 2. Setup Generation (Backstage)

For each ticker:

- **Regime Gating**: Run regime detector → pass only if regime matches strategy type (e.g., momentum in trending market).
- **Entry Rules**: 6 standard motifs (momentum, mean-reversion, pullbacks, breakouts, fades, exotic).
- **Exit Rules**: 3 trailing stop variants → 18 combinatorial legs per ticker → 40,320 hypothetical combinations filtered to 3–5 high-probability setups.

Backstage metadata is cached under `historical_data/<ticker>/setup_cache.json`.

### 3. Setup Validation

Each setup is run through a lightweight backtest on the local OHLCV data:

- **Period**: Last 6 months (adjustable in `config.json`).
- **Fees**: 0.1% slippage model included.
- **Metrics**: Profit Factor, Win Rate, Max Drawdown, Sharpe Ratio.

### 4. Scoring

Combines metrics into a composite score (0.0–1.0) weighted as:

```
setup_score = 0.3 * win_rate + 0.3 * profit_factor + 0.2 * (1 - max_drawdown) + 0.2 * regime_fit
```

## Real World Example: STMicroelectronics (STM.MI) March 2024

### Setup Diagram (ASCII)

```
Time: Jan–Mar 2024
Price: 40–45 range → breakout above 44 → pullback → entry at 42.75
Exit: trailing stop triggers at 43.10 (-0.8% loss) or 45.00 (5.2% gain)
```

```json

```

### Why This Strategy Worked

- Regime: Strong trending market (ADX=31).
- Volume: Avoided spikes on breakout.
- Pullback: Entered at better price, reduced trade cost.

## Mathematical Appendix

### Profit Factor & Drawdown Formulas

```
PF = Total_Winning_Trades / Total_Losing_Trades
max_drawdown = max(1 - (Close_trough / Close_peak))
```

### Regime Fit

Binary alignment value:

- `1.0` if strategy style matches regime.
- `0.1` otherwise.
